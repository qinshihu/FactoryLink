## Что это?

**Один EXE-файл, запуск двойным щелчком, без зависимостей, полностью с открытым исходным кодом и бесплатный** — промышленный шлюз сбора данных с устройств.

Разработан для инженеров на производстве, IT-специалистов заводов и небольших системных интеграторов.

## Зачем это создано?

- Коммерческие шлюзы сбора данных стоят более 1000 евро за штуку — мы сделали бесплатный для всех
- Другие open-source проекты требуют Docker, которым цеховые инженеры не умеют пользоваться
- Большинство международных проектов не имеют документации на китайском, что затрудняет решение проблем

## Как использовать?

1. [Скачать `FactoryLink.exe`](https://github.com/qinshihu/FactoryLink/releases/latest)

2. Запустите двойным щелчком (в правом нижнем углу появится значок в системном трее)

![Значок в трее](images/1(2).png)

3. Браузер автоматически откроет страницу конфигурации (по умолчанию `http://localhost:8000`)

![Страница конфигурации](images/1(1).png)

4. Настройте IP ПЛК, таблицу точек и адрес MQTT

![Конфигурация устройства](images/1(4).png)

5. Нажмите «Запустить сбор» — готово!

> Правый клик по значку в трее позволяет: открыть страницу конфигурации, запустить/остановить сбор, выйти из программы.
> 
> Также поддерживается развёртывание через Docker: [Руководство по развёртыванию Docker](#развёртывание-docker)

## Поддерживаемые протоколы

| Протокол | Поддерживаемые модели | Библиотека |
|----------|----------------------|------------|
| Modbus TCP | Все стандартные устройства Modbus TCP | pymodbus 3.x |
| Modbus RTU | Все стандартные устройства Modbus RTU (последовательный порт) | pymodbus 3.x |
| Siemens S7 | S7-1200 / S7-1500 / S7-300 / S7-400 | python-snap7 3.0 (чистый Python) |
| Mitsubishi MC | FX5U / Серия Q / Серия L | pymcprotocol |

## Основные возможности

- **Один EXE**: Запуск двойным щелчком, не требует среды выполнения
- **Веб-интерфейс**: Настройка через браузер, не требуется знание командной строки
- **Просмотр данных в реальном времени**: Отправка через WebSocket, данные обновляются в реальном времени
- **Пересылка MQTT**: Автоматическая пересылка собранных данных на MQTT-сервер
- **Импорт Excel**: Пакетный импорт конфигураций точек из таблиц Excel
- **Тест подключения**: Проверка связи с ПЛК в один клик
- **Автопереподключение**: Автоматическое переподключение при потере сети с экспоненциальной задержкой
- **Автозапуск**: Настройка автозапуска Windows в один клик
- **Системный трей**: Работа в фоновом режиме, правый клик по значку для операций
- **Просмотр логов**: Просмотр логов сбора непосредственно в веб-интерфейсе с фильтрацией по уровню
- **Горячая перезагрузка**: Нажмите «Применить» после изменений конфигурации для автоматического перезапуска сборщика
- **Автоматическое резервное копирование**: Автоматическое создание `config.json.bak` при каждом сохранении
- **Обработка конфликтов портов**: Автоматическое переключение на 8001, 8002... если порт 8000 занят

## Формат данных

### Топик данных MQTT

```
{topic_prefix}/{device_id}
```

Пример: `factory/gateway-001/dev1`

### Полезная нагрузка MQTT

```json
{
  "gateway": "Шлюз цеха №1",
  "device_id": "dev1",
  "device_name": "Siemens S7-1200",
  "timestamp": 1719123456789,
  "values": {
    "Температура1": {"value": 25.5, "unit": "℃", "quality": "good"},
    "Давление1": {"value": 1.2, "unit": "MPa", "quality": "good"}
  }
}
```

### Топик статуса

```
{topic_prefix}/{device_id}/status
```

```json
{
  "device_id": "dev1",
  "status": "online",
  "message": "Подключение успешно",
  "timestamp": 1719123456789
}
```

- Значения status: `online` (нормальный сбор), `offline` (отключено), `error` (ошибка)
- Значения quality: `good` (нормально), `bad` (ошибка чтения), `uncertain` (подозрительные данные)

## Типы данных точек

| Тип | Описание | Байт |
|-----|----------|------|
| bool | Логический | 1 бит |
| int16 | 16-битное целое со знаком | 2 |
| uint16 | 16-битное целое без знака | 2 |
| int32 | 32-битное целое со знаком | 4 |
| uint32 | 32-битное целое без знака | 4 |
| float | 32-битное с плавающей точкой | 4 |
| double | 64-битное с плавающей точкой | 8 |

> Фактическое значение = Исходное значение × Коэффициент + Смещение. Коэффициент и смещение настраиваются в параметрах точки.

## Файл конфигурации

Все настройки сохраняются в `config.json` в том же каталоге, что и EXE, с автоматическим резервным копированием в `config.json.bak` при каждом изменении.

```json
{
  "gateway_name": "Шлюз цеха №1",
  "devices": [
    {
      "id": "dev1",
      "name": "Siemens S7-1200",
      "protocol": "s7",
      "ip": "192.168.1.100",
      "rack": 0,
      "slot": 1,
      "enabled": true,
      "points": [
        {"name": "Температура1", "address": "DB1.DBD0", "type": "float", "rate": 1.0, "offset": 0.0, "unit": "℃"}
      ]
    }
  ],
  "mqtt": {
    "host": "192.168.1.200",
    "port": 1883,
    "client_id": "gateway-001",
    "topic_prefix": "factory/gateway-001",
    "username": "",
    "password": "",
    "qos": 1,
    "enabled": true
  },
  "collect_interval": 1000,
  "reconnect": {
    "max_retries": 0,
    "base_delay": 1,
    "max_delay": 60
  }
}
```

- `collect_interval`: Интервал сбора в миллисекундах, 1000 = раз в секунду
- `reconnect.max_retries`: 0 = неограниченное количество попыток
- Интервал повтора: экспоненциальная задержка, 1с → 2с → 4с → 8с → 16с → 32с → 60с (предел)

## Форматы адресов протоколов

### Modbus

| Диапазон адресов | Область | Пример |
|------------------|---------|--------|
| 40001-49999 | Регистры хранения | `40001` |
| 30001-39999 | Входные регистры | `30001` |
| 10001-19999 | Дискретные входы | `10001` |
| 00001-09999 | Катушки | `00001` |

### Siemens S7

| Формат | Описание | Пример |
|--------|----------|--------|
| DBx.DBDy | Двойное слово блока DB (32 бита) | `DB1.DBD0` |
| DBx.DBXy.z | Бит блока DB | `DB1.DBX8.0` |
| DBx.DBWy | Слово блока DB (16 бит) | `DB1.DBW0` |
| Mx.y | Бит памяти | `M0.0` |
| Ix.y | Входной бит | `I0.0` |
| Qx.y | Выходной бит | `Q0.0` |

> S7-1200/1500: rack=0, slot=1; S7-300/400: rack=0, slot=2

### Mitsubishi MC

| Формат | Описание | Пример |
|--------|----------|--------|
| Dxxxx | Регистр данных | `D100` |
| Mxxxx | Внутреннее реле | `M100` |
| Xx | Входное реле | `X0` |
| Yx | Выходное реле | `Y0` |
| Wxxxx | Регистр связи | `W100` |

## Формат таблицы точек Excel

| Имя точки | Адрес | Тип данных | Коэффициент | Смещение | Единица |
|-----------|-------|-----------|-------------|----------|---------|
| Температура1 | DB1.DBD0 | float | 1.0 | 0 | ℃ |
| Давление1 | DB1.DBD4 | float | 1.0 | 0 | MPa |
| Состояние работы | DB1.DBX8.0 | bool | 1.0 | 0 | - |

- Первая строка — заголовок (фиксированный формат), данные начинаются со второй строки
- Поддерживаются форматы `.xlsx` и `.xls`
- Скачайте шаблон на странице конфигурации устройства

## Запуск из исходного кода

```bash
# 1. Создать виртуальное окружение (рекомендуется)
python -m venv venv
venv\Scripts\activate

# 2. Установить зависимости Python
pip install -r requirements.txt

# 3. Собрать фронтенд
cd frontend
npm install
npm run build
cd ..

# 4. Запустить бэкенд
cd backend
python main.py
```

Откройте `http://localhost:8000` в браузере.

## Сборка в EXE

```bash
# Убедитесь, что фронтенд собран
cd frontend && npm run build && cd ..

# Запустить скрипт сборки
build.bat
```

Результат: `dist/FactoryLink.exe` (~25 МБ)

## Развёртывание Docker

Если вы не хотите использовать EXE, можно запустить через Docker (подходит для серверов, промышленных ПК, Raspberry Pi).

### Быстрый старт

```bash
# Загрузить образ
docker pull registry.cn-hangzhou.aliyuncs.com/huluwa666/tsq-images-hub:factorylink-latest

# Создать каталоги для конфигурации и логов
mkdir -p /opt/factorylink/{logs,config}

# Запустить контейнер
docker run -d \
  --name factorylink \
  --restart always \
  -p 8000:8000 \
  -v /opt/factorylink/config:/app/config \
  -v /opt/factorylink/logs:/app/logs \
  registry.cn-hangzhou.aliyuncs.com/huluwa666/tsq-images-hub:factorylink-latest
```

### Описание параметров

| Параметр | Описание |
|----------|----------|
| `-p 8000:8000` | Проброс порта веб-страницы конфигурации |
| `-v /opt/factorylink/config:/app/config` | Монтирование каталога конфигурации (здесь хранится config.json) |
| `-v /opt/factorylink/logs:/app/logs` | Монтирование каталога логов |
| `--restart always` | Автоматический перезапуск при сбое контейнера |

### Первый запуск

1. После запуска контейнера откройте `http://ip-вашего-сервера:8000` в браузере
2. Настройте устройства ПЛК и MQTT
3. Нажмите «Запустить сбор»

> Примечание: Версия Docker **не поддерживает** значок в системном трее и автозапуск (функции, доступные только в Windows).

### Использование docker-compose

Создайте `docker-compose.yml`:

```yaml
version: '3'
services:
  factorylink:
    image: registry.cn-hangzhou.aliyuncs.com/huluwa666/tsq-images-hub:factorylink-latest
    container_name: factorylink
    restart: always
    ports:
      - "8000:8000"
    volumes:
      - ./config:/app/config
      - ./logs:/app/logs
```

Запуск:

```bash
docker-compose up -d
```

## Часто задаваемые вопросы

**В: При двойном щелчке по EXE ничего не происходит?**

О: Проверьте системный трей в правом нижнем углу на наличие значка шлюза. Если порт занят, программа автоматически сменит порт. Щёлкните правой кнопкой мыши по значку и выберите «Открыть страницу конфигурации».

**В: Не удаётся подключиться к ПЛК?**

О: Сначала нажмите «Проверить подключение» на странице конфигурации устройства, чтобы проверить IP, порт, номера стойки/слота. Siemens S7-1200/1500 использует rack=0, slot=1; S7-300/400 использует rack=0, slot=2.

**В: MQTT не получает данные?**

О: Проверьте правильность адреса и порта MQTT-сервера, а также что MQTT включён. Просмотрите логи на странице настроек системы для диагностики ошибок подключения.

**В: Как выполнить пакетный импорт точек?**

О: Скачайте шаблон Excel на странице конфигурации устройства, заполните его и импортируйте. Заголовок фиксированный: Имя точки, Адрес, Тип данных, Коэффициент, Смещение, Единица.

**В: Как настроить последовательный порт Modbus RTU?**

О: Введите имя порта напрямую, например `COM3`, `COM4`. Скорость по умолчанию 9600, контроль чётности по умолчанию — Нет (N).

**В: Какой интервал сбора оптимален?**

О: По умолчанию 1000 мс (1 секунда). Для быстрых ПЛК можно установить 500 мс. Для медленных устройств рекомендуется 2000 мс+. Слишком короткий интервал может вызвать тайм-ауты чтения.
